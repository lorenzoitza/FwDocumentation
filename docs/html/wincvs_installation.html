3. SSH

3.1. PuTTY Installation

  • Run the putty-0.59-installer.exe

  • Click "Yes"

  • Click "Next"

  • Change the install path to "C:\utils\putty" (document assumes this is the install location)

  • Click "Next"

  • Click "Next"

  • Click "Install"

  • Click "Finish"

3.2. PuTTYGen Key Generation

  • Start puttygen.exe and select the "SSH2 DSA option"

  • Click on Generate and move the move the mouse around while the key is being generated (part of the key generation process)

  • Once completed the screen will be similar to the above window.

  • Right-Click in the blue highlighted public key area and if "Select All" is available select it as it means that not all the public key is selected. If "Select All" is not available that means that all is well and all the key is selected.

  • Right-Click in the blue highlighted public key area and select "Copy" from the pop-up menu. Then paste it into the email body of an email message to be sent to the CVS administrator who will need this. You will not be able to connect until the administrator installs your public key at his CVS server end.

  • Enter a password of your liking in "Key passphrase" and in "Confirm passphrase"

  • Click on "Save private key" and put the file in an easily accessible location (because Pageant.exe will require it every time it starts up). A good location is C:\utils\putty as that is the default folder that appears when Pageant gives you a private key file open dialog. The document assumes this is the location of the saved private key.

  • Quit the PuTTY key generator

3.3. PuTTY Configuration

We now make sure that PuTTY can connect you to a shell prompt on the server.

  • Start PuTTY. You are presented with a configuration dialog, divided into panels which are navigated by the tree control in the left pane.

  • At "Connection:Data" set the following parameter:

    • Auto-login username = assigned username

  • At "Connection - SSH - Auth" set the following parameter:

    • Private key file for authentication = C:\utils\putty\wycliffe_canada_private.ppk

  • At "Session" set the following parameters:

    • Host Name = cvs.wycliffe.ca

    • Saved Sessions = CVS Canada

  • Click on "Save".

  • Double-clicking on "CVS Canada" or selecting it and clicking "Open" should result in the following window if your setup is correct and your public key has been stored at the CVS server end. At the "Passphrase for key" prompt enter the passphrase you stored with your private key.

3.4. Pageant Configuration

  • Start pageant.exe and it will show up in the Windows tray (an icon displayed as a computer with a hat on).

  • Right-click on the pageant tray icon and select "Add Key"

  • Select C:\utils\putty\wycliffe_canada_private.ppk and open it.

  • Enter your password passphrase you save with your private key earlier and select "OK"

  • Optionally you can put a shortcut to pageant.exe in your "Startup" folder or copy the following into a text file called pageant.reg, save it and then double-click on it to store the registry information in your PC (note that the registry location I chose you need normally to be an administrator of your PC in order for the registry value to be stored):

    REGEDIT4

    [HKEY_LOCAL_MACHINE\SOFTWARE\Microsoft\Windows\CurrentVersion\Run]

    "pageant"="C:\\utils\\putty\\pageant.exe"

  • Optionally you can download the following registry file and double-click on it: pageant_startup.reg

Note

This is just to allow you access to the key, and is not transmitted to the remote system. It is important for our security that you use a good key passphrase. You will need to add you private key and supply your passphrase everytime you enter into Windows and start Pageant.

3.5. Test SSH Using PuTTY

  • Start PuTTY and at the opening window double-click on "CVS Canada" in the "Saved Sessions" list.

  • The following screen should appear without prompting for a keyphrase. This requires that pageant.exe is running and has your private key added (done in an earlier step).

5.2. CVSNT Installation

CVSNT is needed for the protocols. The server itself isn't installed.

  • Start the CVSNT installer and click Next at the "Setup - CVSNT" "Welcome" window.

  • Accept the license agreement and click Next at the "License" window.

  • Select "Custom".

  • At the "Custom Setup" window set the install path to "C:\cvsnt "using the "Browse" button, select at least the following items and click Next.

    • Uncheck the RCS Wrappers option

    • Uncheck the CVSNT Server option

    • Workspace Viewer

    • U3 Support

    • Documentation

    • External Command (:ext:) protocol

    • SSH Client (:ssh:) protocol

    Note

    Only the SSH Client protocol is required but the other selections provide alternative methods of using SSH.

    Leaving the default options for the Protocols is the best thing to do.

  • Click Install.

  • Click Finish at the following window.

5.3. WinCVS Configuration

  • Launch WinCvs.

    Note

    Even though Windows isn't case-sensitive CVS is case-sensitive and therefore pay attention to the case of the various entries.

  • Close the "Tip of the Day" window.

  • If you didn't install Python earlier then then click Ok at the "Install Python" window. Python isn't required for basic use of WinCVS but can be installed separately if full use of WinCVS is desired.

  • At the "WinCVS Preferences" dialog window select the Globals tab and set the following parameters:

    • Supply control when adding files = checked

    • Compression level = checked and set to 9

    • Prune empty directories = checked

    • Disable splash screen = checked

  • This assumes that Python 2.5 was installed in "C:\Python25" and that TCL installed as part of Python is going to be used. At the "WinCVS Preferences" dialog window select the WinCvs tab and set the following parameters:

    • Python DLL = python25.dll

    • TCL DLL = C:\Python25\DLLs\tcl84.dll

  • Select the CVS tab and set the "HOME" parameter to "C:\Program Files\GNU\WinCvs 2.0" or whatever the install directory path is. It can be set to just about any directory but generally this is the preferred location.

  • Click on "OK"

  • Create the following directory "C:\VCS\Wycliffe\projects". The case of the path is important to be the same as specified because the CVS repository is case-sensitive. The instructions presented here will permit you to do a complete checkout of the entire repository at a later date. For now these instructions will be limited to what is the basic structure for the FieldWorks files.

  • At the main WinCVS window select "View -Browse Location - Change"

  • Select the previously created "C:\VCS\Wycliffe\projects" location

  • At the toolbar location box highlight but DON'T click on "C:\" and then press on the "Delete" key on your keyboard (this is counter-intuitive!).

    Note

    If you inadvertently clicked on "C:\" you will have to first reselect "C:\VCS\Wycliffe\projects" before you will be allowed to delete "C:\" as just previously described.

  • At the remove confirmation prompt click "Yes"

  • Right-Click on the "projects" folder on the left-hand window pane and select "Checkout module" from the pop-up menu.

  • At the "Checkout Settings" dialog window set the following properties:

    • CVSROOT = :ssh;username=blouiny;hostname=cvs.wycliffe.ca:/var/lib/cvs/mdu/

      Note

      Replace the username value with your own assigned username.

      You can set CVSROOT using the "..." button beside the "CVSROOT" text box to set this field in a more interactive manner

    • Module name and path on the server = FieldWorks

      Note

      You can set path using the "..." button beside the "Module name ..." text box if the CVSROOT is properly set in the "Select Module" window or you can simply enter it manually. It is case sensitive. If using the "Select Module" window after setting the CVSROOT value then press the "Refresh" button.

    • Local folder to checkout to = C:\VCS\Wycliffe\projects
