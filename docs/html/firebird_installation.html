1. Introduction

The installation and setup of Firebird consists of these main steps:

  1. Installing the Firebird files
  2. Installing the FlameRobin application
  3. Creating a Firebird database container
  4. Install custom UDF DLLs

3. Firebird Database Server Installation

  • Ensure that you've uninstalled any already installed Firebird system.

  • Launch the Firebird-2.0.1.12855-1-Win32.exe setup file.

  • Click "OK".

  • Click "Next".

  • Read the license agreement, select "I accept the agreement" and click "Next".

  • Click "Next".

  • Change the install directory to "C:\Firebird" as shown above.

  • In Windows you have to choose the "Super Server" option. "Super Server" is the preferred platform in general. Select the "Server components" and "Developer and admin tools components" and then click "Next".

  • Modify the start program menu item so that it doesn't have the version number in it by naming it "Firebird".Click "Yes".

  • At the "Select Additional Tasks" Window set the following properties:

    • Use the Guardian to control the server = checked

    • Run as a Service = checked

    • Start Firebird automatically everytime you boot up = checked

    • Copy Firebird client library to system directory = checked

    • Create client library as GDS32.DLL for legacy app. support = checked

    • Install Control Panel Applet (if present) = checked

  • Click "Next".

  • Click "Install".

  • Click "Next".

  • Select "Start Firebird Service Now" and click "Finish".

  • This completes the basic Firebird files installation.

  • Verify using the Services Control Panel (shown below) that the Firebird Guardian and Firebird Server are shown as running. Only the Firebird Guardian service needs to be set to automatic if you want Firebird to startup on system startup. If not set both services to manual.

    Note
    The services control panel on Windows 2000 can be found by following this path Start - Settings - Control Panel - Administrative Tools - Services or for Win2k and WinXP selecting Start - Run and entering "services.msc /s" at the prompt.

  • Verify in the event viewer application log (shown below) for the following message with no errors in either the application log or system log related to Firebird.

    Note
    The services control panel on Windows 2000 can be found by following this path Start - Settings - Control Panel - Administrative Tools - Event Viewer or for Win2k and WinXP selecting Start - Run and entering "eventvwr.msc /s" at the prompt.

  • Create the following folder structure: "C:\Firebird\logs"

5. Firebird Database Container Creation

  • Modify Password for Sysdba:

    • Go into a command window under C:\firebird\bin

    • At the shell (Windows or Unix) prompt type the following:

      • gsec –user sysdba –password masterkey

      A "GSEC>" prompt should now appear as your now logged into the database using the default username/password that is used for newly created databases.

      At the "GSEC>" prompt type the following:

      • modify sysdba –pw inscrutable

      • quit

  • Create the following folder structure: "C:\Firebird\data\fieldworks".

  • Start FlameRobin.

  • From the "Database" menu select "Create new database...".

  • Enter the following values:

    • Display name: FieldWorks

    • Database path: C:\Firebird\data\fieldworks\fw01.db

    • Username: sysdba

    • Password: inscrutable

    • Page size: 8192

    • Charset: NONE

    • Role: (leave empty)

    • SQL Dialect: Dialect 3

    Note
    The selected Charset is NONE. Do not use UNICODE_FSS as this will cause problems when registering UDFs with CSTRING(32767) parameters. Database can still be used with the Unicode character set even if it was set with a default of NONE. The OCTETS setting was tried and this turned out to have it's own set of problems when trying to use Unicode_FSS parameters in stored procedures, etc.

  • FlameRobin will now display a "FieldWorks" icon in it's tree list.

5.2. FlameRobin Miscellaneous Items

  • To connect to the database start FlameRobin and double-click on the "Localhost" icon followed by a double-click on the "FieldWorks" icon.

  • Use the following preference settings when desiring to do some file logging such as when wanting to debug an issue and send some log files to another developer for help. The "Preferences" window is accessed via the "Preferences" item in the "View" menu.

    • Log DML statements: checked

    • Enable logging to file: checked

    • Log file name: C:\Firebird\logs\flamerobin.log

    • Add header to each statement

    • Add SET TERM when different terminator is used in editor
