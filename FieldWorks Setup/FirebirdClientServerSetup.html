Firebird Client/Server Setup

For Windows

Firebird Setup

1.      Download the Firebird installer from Firebird's download page. As of this writing, the current version is 2.1.2. Get the “Windows executable installer for Classic or Superserver, recommended for first-time users.”

2.      Make sure you have uninstalled any existing Firebird system.

3.      Launch the installer.

4.      Go through the welcome screen and the license agreement.

5.      Selecte a destination directory. So far the default works fine for me.

6.      You'll see a “Select Components” dialog. In Windows we must choose the “Super Server” option, and this is said to be the preferred platform in general. Also check/tick the box for “Developer and admin tools components”.

7.      Select a Start Menu folder. I prefer to take the version number off.

8.      In the “Select Additional Tasks” dialog:

1.      Use the Guardian to control the server: checked/ticked.

2.      Run as a Service: checked/ticked.

3.      Start Firebird automatically every time you boot up: checked/ticked.

4.      Copy Firebird client library to system directory: checked/ticked.

5.      Create client library as GDS32.dll for legacy app. support: checked/ticked.

6.      Install Control Panel Applet (if present): checked/ticked.

Windows Vista CAUTION
If you are installing onto Windows Vista, the installer option to install the Control Panel applet must be DISABLED to avoid having it break the Control Panel on your Vista system.

9.      Click through the next screens and click on Install and Finish.

10.  You can verify that Firebird installed correctly by going to the Services Control Panel. (Start/Settings/Control Panel/Administrative Tools/Services -or- Start/Run/”services.msc /s” at the prompt.) Firebird Guardian and Firebird Server should be shown as running. Only the Firebird Guardian service needs to be set to automatic if you want Firebird to start up on system start up. If not, set both services to manual.

11.  Change the password for the system DBA user sysdba:

1.      Go to the directory where you installed Firebird, and then go to the \bin subdirectory.

2.      Open a command window.

3.      Type the following to get into the Firebird gsec utility: “gsec -user sysdba -password masterkey” A “GSEC>” prompt will appear.

4.      Type “modify sysdba -pw inscrutable”.

5.      Type “quit”.

FlameRobin Setup

FlameRobin is a lightweight, cross-platform, open-source database administration tool for Firebird. You don't need it to run tests, but it can be handy for working with Firebird data.

1.      Download the FlameRobin installer from its Sourceforge download page. As of this writing, the current version is 0.9.2.

2.      Launch the installer.

3.      Go through the welcome screen and license dialog.

4.      Select a start menu folder.

5.      Decide if you want a desktop icon and/or a Quick Launch icon.

6.      Click Install.

The first time you launch FlameRobin, you will be presented with a dialog tthat complains about a missing configuration file. This is normal for first-time users. (The FlameRobin configuration file, fr_databases.conf, is useful to restore connections, among other things.)

If you want to look at a Firebird database, you will first need to register it. Go to Database/Register existing database...

If you use FlameRobin to create a new database, you can likely use the UTF8 character set. We had started to work with that during the old port project. The character set NONE also stored Unicode data. Do not use the UNICODE_FSS character set. It has problems. We were also using a page size of 8192.
