MySQL Client/Server Setup

For Windows

MySQL Setup

1.      Download the MySQL installer. The installer is at MySQL's download page. As of this writing, the current version is 5.1 GA. Get the “Microsoft Windows 32. (Windows Installer format)”, assuming you have a 32-bit machine.

2.      Launch the installer.

3.      You can run the Typical setup, but then you will miss the Developer Components: C Include files/Lib files; Embedded Server; and Scripts and Examples. I suggest (for now, anyway) that you run the Complete setup and select those.

4.      Skip the ads for MySQL Enterprise and the Monitor Service.

5.      When the wizard completes, leave the Configure MySQL Server now checked/ticked. You can register MySQL if you really want to.

1.      Select the Detailed Configuration.

2.      Select the Developer Machine.

3.      Select Multifunctional Database. (MySQL has two storage engines. MyISAM appears to be what we want for this system, but it's hard to know until both have been tried. See this link for more information.)

4.      Select the default for tablespace settings.

5.      For Concurrent Users, select Manual Setting, and enter 2. (This is an approximate number of concurrent connections. We could often have 1, but we could also have TE, FLEx, and FLEx's parser running. So I picked 2.)

6.      Leave TCP/IP Networking and Strict Mode enabled.

7.      Select “Best Support for Multilingualism”, which is UTF8.

8.      Install as a Windows Service, and Include the Bin directory in the Windows PATH.

9.      Assuming that you do not have MySQL installed already, leave the Current root password blank. Use “inscrutable” for the new root password. Do not create an anonymous account.

10.      Press Execute and Finish.

11.      Find MySQL in Services and stop MySQL. Find the My.ini configuration file. This should be in your MySQL installation file, such as C:\Program Files\MySQL\MySQL Server 5.1\my.ini. (Or MySQL Server 6.0.11\my.ini.) Add this line to the bottom of the file: max_allowed_packet=1GB. (If you add the line to the top of the file, MySQL will not start back up.) Restart MySQL.

12.      (NOTE: When I did this, MySQL wouldn't run proprely. I have not done this step. That means data gets sent to the default datadir: "C:/Documents and Settings/All Users/Application Data/MySQL/MySQL Server 5.1/Data/". Should MySQL become our database of choice we will have to return to this issue.) Find the My.ini configuration file. This should be in your MySQL installation file, such as C:\Program Files\MySQL\MySQL Server 5.1\my.ini. Set datadir to “C:/fw<whatever>/output/debug”.

13.      This caution is on the MySql installation instructions, but I haven't figured out how to configure Norton yet. I have not yet been flagged:

Using virus scanning software such as Norton/Symantec Anti-Virus on directories containing MySQL data and temporary tables can cause issues, both in terms of the performance of MySQL and the virus-scanning software mis-identifying the contents of the files as containing spam. This is because of the fingerprinting mechanism used by the virus scanning software, and the way in which MySQL rapidly updates different files, which may be identified as a potential security risk.

After installing MySQL Server, it is recommended that you disable virus scanning on the main directory (datadir) being used to store your MySQL table data. There is usually a system built into the virus scanning software to allow certain directories to be specifically ignored during virus scanning.

In addition, by default, MySQL creates temporary files in the standard Windows temporary directory. To prevent the temporary files also being scanned, you should configure a separate temporary directory for MySQL temporary files and add this to the virus scanning exclusion list. To do this, add a configuration option for the tmpdir parameter to your my.ini configuration file. For more information, see Section 2.3.7, “Creating an Option File”.

14. To get the Rename Database tests to pass, you may need to logout and log in again. This seems to have something to do with updates to path variables taking effect.

For Linux

Install the package mysql-server.

If installing the package doesn't ask for a password for root, then you have to reconfigure mysql-server:

sudo dpkg-reconfigure mysql-server

If asked for a password for root enter 'inscrutable'.

See also Build_FuildWorks wiki page.
